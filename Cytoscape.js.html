关于

Cytoscape.js是一个开源的图形理论（又名网络）写在JS库。您可以使用Cytoscape.js进行图形分析和可视化。

Cytoscape.js允许您轻松地显示和操作丰富的交互式图形。因为Cytoscape.js允许用户与图形交互，并且库允许客户端钩入用户事件，Cytoscape.js很容易集成到您的应用程序中，特别是因为Cytoscape.js同时支持桌面浏览器（如Chrome）和移动浏览器，就像在iPad上。Cytoscape.js包括所有你想要的开箱即用的手势，包括缩放，缩放，框选择，平移等。

Cytoscape.js也有图表分析：图书馆在图论中包含许多有用的功能。您可以使用Cytoscape.js无头地在Node.js上在终端或Web服务器上进行图分析。

Cytoscape.js是一个开源项目，任何人都可以自由贡献。欲了解更多信息，请参阅GitHub的自述。

该库是在创建唐纳利中心在多伦多大学。它的继任者Cytoscape的网页。

符号

图模型

Cytoscape.js支持许多不同的图论理论。它支持有向图，无向图，混合图，循环，多图，复合图（一种超图）等。

我们定期进行添加，并增强了图书馆，我们很乐意接受的功能要求和拉请求。

架构和API

在架构中有两个组件，开发人员需要关心自己为了使用Cytoscape.js，核心（即图实例）和集合。在Cytoscape.js中，核心是开发人员进入库的主入口点。从核心，开发人员可以运行布局，改变视口，并在图表上执行其他操作作为一个整体。

核心提供了几个函数来访问图形中的元素。这些函数中的每一个都返回一个集合，图形中的一组元素。集合上提供了函数，允许开发人员过滤集合，对集合执行操作，遍历关于集合的图，获取集合中元素的数据等。

请注意，默认情况下集合是不可变的，这意味着集合中的元素集合不能更改。API在必要时返回具有不同元素的新集合，而不是更改现有集合。这允许开发人员安全地对集合使用集合理论操作，在功能上使用集合，等等。注意，因为集合只是元素的列表，所以创建新集合是便宜的。

功能

有几种类型可以执行不同的函数，并且在文档中用于表示这些类型的变量名称概述如下：

速记作品
cy核心
eles一个或多个元素（节点和边）的集合，
ele单个元素（节点或边）的集合
nodes一个或多个节点的集合
node一个单个节点的集合
edges一个或多个边缘的集合
edge一个单一边缘的集合
layout布局
ani动画

默认情况下，函数返回一个返回参考调用对象，以允许链接（例如obj.fn1().fn2().fn3()）。除非本文档中另有说明，否则函数可以以此方式链接，除非指定了不同的返回值。这适用于核心和集合。

该返回值的函数，请注意，调用一个奇异- ，ele，node或edge-在一个以上的元素的集合函数将返回预期的值仅用于第一个元素。

对象所有权

当将对象传递给Cytoscape.js用于创建元素，动画，布局等时，对象被认为由Cytoscape拥有。像元素之类的对象有几个级别，并且每次传递给Cytoscape时都会对这些对象执行深层副本，从而产生额外的费用。如果需要，dev可以在将对象传递给Cytoscape之前手动复制对象。然而，大多数开发人员大多数时间不需要复制。

手势

Cytoscape.js支持多种手势：

  • 抓取并拖动背景进行平移：触摸和桌面
  • 双指拨放：触摸和桌面（支持触控板）
  • 鼠标滚轮缩放：桌面
  • 两个手指触控板上或下缩放：桌面
  • 点按即可选择：触摸和桌面
  • 点按背景以取消选择：桌面
  • 点按要取消选择的背景：桌面和触摸
  • 通过修改键（shift，命令，控制，alt）多个选择+点击：桌面
  • 框选择：触摸（三指滑动）和桌面（修改键+ mousedown然后拖动）
  • 抓取和拖动节点：触摸和桌面

所有手势动作都可以由dev控制，在需要时打开或关闭它们。

位置

节点的位置是指其身体的中心点。

还有就是要作出地位的重要区别：一个位置可能是一个模型位置或渲染的位置。

模型位置 - 如其名称所示 - 是存储在元素的模型中的位置。元素的模型位置保持不变，尽管有缩放和平移的变化。数值样式属性值在模型坐标中指定，例如，宽度为20px的节点将在缩放1时宽度为20像素。

渲染位置是相对于视口的屏幕上位置。例如，一个呈现的位置{ x: 100, y: 100 }指定点100像素向右和100像素从视口的左上角向下。模型位置和渲染位置在缩放1和平移（0，0）处相同。

元素的呈现位置随着缩放和平移变化而自然地改变，因为元素在视口中的屏幕位置随着应用缩放和平移而改变。平移总是以渲染坐标测量。

在本文档中，除非另有说明，“位置”是指模型位置。

一个节点的位置可以手动设置，也可以自动地使用来设置布局。因为两个节点的位置影响它们之间的边缘的长度，布局有效地设置边缘长度。

元素JSON

给出了用于将元素加载到Cytoscape.js中的元素的大纲格式示例：

cytoscape({

  container: document.getElementById('cy'),

  elements: [
    { // node n1
      group: 'nodes', // 'nodes' for a node, 'edges' for an edge
      // NB the group field can be automatically inferred for you but specifying it
      // gives you nice debug messages if you mis-init elements

      // NB: id fields must be strings or numbers
      data: { // element data (put dev data here)
        id: 'n1', // mandatory for each element, assigned automatically on undefined
        parent: 'nparent', // indicates the compound node parent id; not defined => no parent
      },

      // scratchpad data (usually temp or nonserialisable data)
      scratch: {
        foo: 'bar'
      },

      position: { // the model position of the node (optional on init, mandatory after)
        x: 100,
        y: 100
      },

      selected: false, // whether the element is selected (default false)

      selectable: true, // whether the selection state is mutable (default true)

      locked: false, // when locked a node's position is immutable (default false)

      grabbable: true, // whether the node can be grabbed and moved by the user

      classes: 'foo bar' // a space separated list of class names that the element has
    },

    { // node n2
      data: { id: 'n2' },
      renderedPosition: { x: 200, y: 200 } // can alternatively specify position in rendered on-screen pixels
    },

    { // node n3
      data: { id: 'n3', parent: 'nparent' },
      position: { x: 123, y: 234 }
    },

    { // node nparent
      data: { id: 'nparent', position: { x: 200, y: 100 } }
    },

    { // edge e1
      data: {
        id: 'e1',
        // inferred as an edge because `source` and `target` are specified:
        source: 'n1', // the source node id (edge comes from this node)
        target: 'n2'  // the target node id (edge goes to this node)
      }
    }
  ],

  layout: {
    name: 'preset'
  },

  // so we can see the ids etc
  style: [
    {
      selector: 'node',
      style: {
        'content': 'data(id)'
      }
    },

    {
      selector: ':parent',
      style: {
        'background-opacity': 0.6
      }
    }
  ]

});

复合节点

复合节点是传统图形模型的补充。复合节点包含多个子节点，类似于HTML DOM元素可以如何包含多个子元素。

复合节点通过指定的parent一个元素的领域data。类似于source和target边缘领域，parent领域是不变的：可以当节点添加到图中可以指定一个节点的父节点，并在该点之后，这种亲子关系是不可改变的。但是，您可以通过有效的移动子节点eles.move()。

复合父节点不具有独立的维度（位置和大小），因为这些值由后代节点的位置和维度自动推断。

至于API而言，复合节点被视为就像普通节点-除了在明确复合功能一样node.parent()。这意味着，传统的图论功能，如eles.dijkstra()并eles.neighborhood()没有作出复合节点特殊津贴，所以你可能需要作出具体取决于您的用例的API不同的呼叫。

例如：

var a = cy.$('#a'); // assume a compound node

// the neighbourhood of `a` contains directly connected elements
var directlyConnected = a.neighborhood();

// you may want everything connected to its descendants instead
// because the descendants "belong" to `a`
var indirectlyConnected = a.add( a.descendants() ).neighborhood();

Cytoscape.js的实例对应于图形。您可以按如下所示创建实例：

var cy = cytoscape({
  container: document.getElementById('cy') // container to render in
});

你可以传递一个jQuery的实例作为container方便：

var cy = cytoscape({
  container: $('#cy')
});

如果您在运行的Node.js或Cytoscape.js否则运行它headlessly，你不会指定container选项。在隐式无头环境（如Node.js）中，实例自动无头。要明确运行无头实例（例如，在浏览器），您可以指定options.headless为true。

指定基本选项

对于可视化的container，elements，style，和layout选项通常应设置：

var cy = cytoscape({

  container: document.getElementById('cy'), // container to render in

  elements: [ // list of graph elements to start with
    { // node a
      data: { id: 'a' }
    },
    { // node b
      data: { id: 'b' }
    },
    { // edge ab
      data: { id: 'ab', source: 'a', target: 'b' }
    }
  ],

  style: [ // the stylesheet for the graph
    {
      selector: 'node',
      style: {
        'background-color': '#666',
        'label': 'data(id)'
      }
    },

    {
      selector: 'edge',
      style: {
        'width': 3,
        'line-color': '#ccc',
        'target-arrow-color': '#ccc',
        'target-arrow-shape': 'triangle'
      }
    }
  ],

  layout: {
    name: 'grid',
    rows: 1
  }

});

很常用的选项

container：应在其中呈现图形的HTML DOM元素。这是未指定的，如果Cytoscape.js无头运行。

elements：数组指定为普通对象的元素。为了方便起见，可以将此选项指定为解析为元素JSON的promise。

style：该样式表用于设计图形。为了方便起见，可以将此选项指定为解析到样式表的promise。

layout：指定布局选项的纯对象。其中布局最初运行由指定的name领域。指一个布局的文档，它支持的选项。如果你想指定的节点位置自己在你的元素JSON，您可以使用preset布局-默认情况下它不设置任何职位，让你的节点的当前位置（如在指定options.elements的初始化时间）。

初始视口状态

zoom：图形的初始缩放级别。确保禁用视操作选项，比如fit，在你的布局，使应用的布局时，它不覆盖。您可以设置options.minZoom并options.maxZoom设置变焦水平的限制。

pan：图的初始平移位置。确保禁用视操作选项，比如fit，在你的布局，使应用的布局时，它不覆盖。

互动选项

minZoom：图形缩放级别的最小边界。视口不能缩放为小于此缩放级别。

maxZoom：图形缩放级别的最大边界。视口不能缩放到大于此缩放级别。

zoomingEnabled ：是否通过用户事件和编程方式启用图形缩放。

userZoomingEnabled：是否允许用户事件（例如鼠标滚轮，缩放到缩放）放大图形。对缩放的程序化更改不受此选项的影响。

panningEnabled ：是否通过用户事件和编程方式启用图形的平移。

userPanningEnabled：是否允许用户事件（例如拖动图形背景）平移图形。对平移的编程更改不受此选项的影响。

boxSelectionEnabled：是否启用框选择（即拖动框覆盖，并释放它选择）。如果启用，用户必须点击才能平移图形。

selectionType：表示来自用户输入的选择行为的字符串。为'additive'，由用户做出的新的选择增加了组当前选定元素。为'single'，由用户作出新的选择变得整组目前所选元素的（即以前的元素是未选择的）。

touchTapThreshold＆desktopTapThreshold：一个非负整数，指示用户可能抽头手势期间分别移动，在触摸设备和桌面设备所允许的最大距离。这使得用户更容易敲击。这些值具有固定的默认值，因此建议不要更改这些选项，除非您有非常好的理由这样做。更大的值几乎肯定会有不良后果。

autoungrabify：节点是否应ungrabified（而不是由用户grabbable）默认情况下（如true，覆盖单个节点状态）。

autolock：是否节点应该在默认情况下被锁定（不可拖动都）（如true，覆盖单个节点状态）。

autounselectify：是否节点应该是默认unselectified（不可变的选择状态）（如true，覆盖单个元素状态）。

渲染选项

headless：一个方便的选项，初始化实例无头运行。你不需要在隐式无头的环境中设置它（例如Node.js）。然而，这是很方便的设置headless: true，如果你想在浏览器中无头的实例。

styleEnabled：一个布尔值，指示是否应使用样式。对于无头（即在浏览器外部）环境，显示不是必需的，因此样式也不必要 - 从而加速您的代码。如果在特殊情况下需要，可以在无头环境中手动启用样式。请注意，如果计划呈现图形，则禁用样式没有意义。

hideEdgesOnViewport：呈现提示，当设置为true使得渲染，而视口被操纵不呈现边缘。这使得平移，缩放，拖动等对大图形更加敏感。由于性能增强，此选项现在大部分无效。

textureOnViewport：渲染提示，当设置为true使得渲染平移和缩放，而不是绘制元素，使得大图更适应过程中使用一个纹理。由于性能增强，此选项现在大部分无效。

motionBlur：呈现提示：被设置为当true使渲染器使用运动模糊效果，使帧之间的过渡更平滑似。这可以增加大图的感知性能。由于性能增强，此选项现在大部分无效。

motionBlurOpacity：当motionBlur: true，此值控制运动模糊帧的不透明度。更高的值使运动模糊效果更显着。由于性能增强，此选项现在大部分无效。

wheelSensitivity：在变焦时更改滚轮灵敏度。这是乘法修饰符。因此，0和1之间的值会降低灵敏度（变焦速度变慢），大于1的值会增加灵敏度（变焦速度变快）。

pixelRatio：覆盖了手动设定值屏幕的像素比例（1.0推荐，如果已设置）。这可以用于通过减少需要呈现的有效区域来提高高密度显示器的性能，尽管在最近的浏览器版本中这不是必需的。如果你想使用的硬件的实际像素比，可以设置pixelRatio: 'auto'（默认）。

        细节

        通常，当修改元素时，每个修改可以触发样式计算和重绘 - 这取决于重绘的时间。例如，以下将导致两个样式计算和至少一个绘图：

  // at least 1 redraw here
  // possibly 3 total depending on speed of above operations
  // (for one ele almost certainly 1 redraw, but consider many eles)
;

        这对于少数元素的少数操作不是问题，但是对于许多元素上的许多操作，最终得到冗余样式计算和可能的多余重绘。在最坏的情况下，你有eles.length * numOps风格的更新和重绘-两者款式更新，并重绘可能很昂贵。在使用时，最坏的情况下cy.batch()，您限制款式更新到eles.length你限制重绘到只有一个。

        • 您不能可靠地读取元素样式或尺寸（可能已更改，或计算的值可能已过期），
        • 你可能不希望使用eles.style()等等，因为他们强迫式的旁路，而不是重新计算。

  cy.$('#j')
    .data('weight', '70')
    .addClass('funny')
    .removeClass('serious')
  ;
});

cy.$('#j')

                                                                                          eles.breadthFirstSearch（）      
                                                                                          别名： eles.bfs()，

                                                                                          在集合中的元素中执行宽度优先搜索。

                                                                                          eles.breadthFirstSearch（， 选项）
                                                                                          • 选项
                                                                                            • 根

                                                                                              根节点（选择器或集合）开始搜索。

                                                                                            • 访问：功能（I，深度，V，E，U） [可选]

                                                                                              当在搜索中访问节点时调用的处理程序函数。该处理程序返回true时，发现所需的节点，并返回false到取消搜索。

                                                                                              • 一世

                                                                                                指示此节点的索引是第i个被访问节点。

                                                                                              • 深度

                                                                                                这个节点离根节点有多少边跳。

                                                                                              • v

                                                                                                当前节点。

                                                                                              • e

                                                                                                将前一个节点连接到当前节点的边。

                                                                                              • u

                                                                                                上一个节点。

                                                                                            • 定向 [可选]

                                                                                              一个布尔值，指示是否算法应该沿着边缘只去从源到目标（默认false）。

                                                                                          细节

                                                                                          请注意，此函数仅对调用集合中的图的子集执行宽度优先搜索。

                                                                                          该函数返回一个包含两个集合（对象{ path: eles, found: node }），由搜索和搜索的路径中找到的节点：

                                                                                          • 如果没有找到节点，则found是空的。
                                                                                          • 如果你的处理函数返回false，那么唯一的路径到这一点被返回。
                                                                                          • 返回的路径包括边缘，使得如果path[i]是一个节点，则path[i - 1]是用于获取该节点的边缘。

                                                                                          例子

                                                                                          var bfs = cy.elements().bfs({
  roots: '#e',
  visit: function(i, depth){
    console.log( 'visit ' + this.id() );

    // example of finding desired node
    if( this.data('weight') > 70 ){
      return true;
    }

    // example of exiting search early
    if( this.data('weight') < 0 ){
      return false;
    }
  },
  directed: false
});

var path = bfs.path; // path to found node
var found = bfs.found; // found node

// select the path
path.select();

    console.log( 'visit ' + this.id() );

    if( this.data('weight') > 70 ){
      return true;
    }

    // example of exiting search early
    if( this.data('weight') < 0 ){
      return false;
    }
  },
  directed: false
});

var found = dfs.found; // found node

// select the path

                                                                                          选择器

                                                                                          注意和注意事项

                                                                                          选择器的功能类似于DOM元素上的CSS选择器，但Cytoscape.js中的选择器处理图形元素的集合。需要注意的是无论选择器可以被指定作为参数的函数，一个eles.filter()风格的过滤功能可替代选择的使用。例如：

                                                                                          cy.$('#j').neighborhood(function(){
  return this.isEdge();
});

                                                                                          选择器可以组合在一起，在Cytoscape.js中进行强大的查询，例如：

                                                                                          // get all nodes with weight more than 50 and height strictly less than 180
cy.elements("node[weight >= 50][height < 180]");

                                                                                          选择器可以用逗号连接在一起（有效地创建逻辑OR）：

                                                                                          // get node j and the edges coming out from it
cy.elements('node#j, edge[source = "j"]');

                                                                                          重要的是要注意，字符串需要用引号括起来：

                                                                                          //cy.filter('node[name = Jerry]'); // this doesn't work
cy.filter('node[name = "Jerry"]'); // but this does

                                                                                          请注意，元字符（^ $ \ / /（）|？+ * [] {}，。）需要转义：

                                                                                          cy.filter('#some\\$funky\\@id');

                                                                                          组，类和ID

                                                                                          node，edge或*（组选）匹配基于族元素（node为节点edge的边缘，*所有）。

                                                                                          .className匹配具有指定类（如使用元素.foo的一个名为“foo”类）。

                                                                                          #id匹配元件具有匹配的ID（例如，#foo相同[id = 'foo']）

                                                                                          数据

                                                                                          [name]
                                                                                          匹配的元素，如果他们有定义的指定数据属性，即不undefined（例如[foo]对于名为“foo”的属性）。这里，null被认为是一个确定的值。

                                                                                          [^name]
                                                                                          匹配的元素，如果没有定义指定数据属性，即undefined（例如[^foo]）。这里，null被认为是一个确定的值。

                                                                                          [?name]
                                                                                          匹配的元素，如果指定的数据属性是truthy值（例如[?foo]）。

                                                                                          [!name]
                                                                                          匹配的元素，如果指定的数据属性是falsey值（例如[!foo]）。

                                                                                          [name = value]
                                                                                          匹配的元素，如果他们的数据属性的特定值（如火柴[foo = 'bar']或[num = 2]）。

                                                                                          [name != value]
                                                                                          匹配的元素，如果他们的数据属性不符合规定值（例如[foo != 'bar']或[num != 2]）。

                                                                                          [name > value]
                                                                                          匹配元件，如果他们的数据属性是比规定值（例如，大于[foo > 'bar']或[num > 2]）。

                                                                                          [name >= value]
                                                                                          匹配元件，如果他们的数据属性是否大于或等于规定值（例如，[foo >= 'bar']或[num >= 2]）。

                                                                                          [name < value]
                                                                                          匹配元件，如果他们的数据属性是小于规定值（例如[foo < 'bar']或[num < 2]）。

                                                                                          [name <= value]
                                                                                          匹配元件，如果他们的数据属性小于或等于规定值（例如，[foo <= 'bar']或[num <= 2]）。

                                                                                          [name *= value]
                                                                                          匹配的元素，如果他们的数据属性包含指定的值作为一个子字符串（例如[foo *= 'bar']）。

                                                                                          [name ^= value]
                                                                                          匹配的元素，如果他们的数据属性与指定的值（例如启动[foo ^= 'bar']）。

                                                                                          [name $= value]
                                                                                          如果他们的数据属性与指定的值（例如结束匹配元素[foo $= 'bar']）。

                                                                                          @（数据属性运营商修改）
                                                                                          前面加上运营商，这样是不区分大小写（例如[foo @$= 'ar']，[foo @>= 'a']，[foo @= 'bar']）

                                                                                          !（数据属性操作者调整值）
                                                                                          预先考虑的操作，以便它被否定（例如[foo !$= 'ar']，[foo !>= 'a']）

                                                                                          [[]]（元数据括号）

                                                                                          样式

                                                                                          Cytoscape.js中的样式尽可能遵循CSS约定。在大多数情况下，属性具有与其对应的CSS名称相同的名称和行为。但是，CSS中的属性不足以指定图形某些部分的样式。在这种情况下，引入了Cytoscape.js独有的其他属性。

                                                                                          为了简单和易用性，特异性规则在样式表被完全忽略。对于给定元素的给定样式属性，最后匹配的选择器将胜出。

                                                                                          格式

                                                                                          在指定的样式初始化可以在函数的格式，在一个普通的JSON格式，或在一个字符串格式-如果你想从服务器拉下风格平原JSON格式的字符串格式是更有益。

                                                                                          字符串格式

                                                                                          请注意，每个属性的尾部分号是必需的。解析肯定会失败，没有他们。

                                                                                          示例样式文件：

                                                                                          /* comments may be entered like this */
node {
  background-color: green;
}

                                                                                          初始化时：

                                                                                          cytoscape({
  container: document.getElementById('cy'),

  // ...

  style: 'node { background-color: green; }' // probably previously loaded via ajax rather than hardcoded

  // , ...
});

                                                                                          纯JSON格式

                                                                                          cytoscape({
  container: document.getElementById('cy'),

  // ...

  style: [
    {
      selector: 'node',
      style: {
        'background-color': 'red'
      }
    }

    // , ...
  ]

  // , ...
});

                                                                                          cytoscape({
  container: document.getElementById('cy'),

  // ...

    .selector('node')
      .style({

      })

      // ...

});

                                                                                          函数值

                                                                                          在JSON或函数样式表格式中，可以将函数指定为style属性的值。以这种方式，可以通过基于每个元素的函数来指定样式值。

                                                                                          在某些情况下，使用函数作为样式属性值可能是方便的。但是，它可能不是一个高级选项。因此，它可能是值得的，如果可以使用缓存，诸如通过使用lodash _.memoize()功能。

                                                                                          注意，如果使用函数作为样式值，则不可能将样式表序列化和反序列化为JSON。

                                                                                          例：

                                                                                          cytoscape({
  container: document.getElementById('cy'),

  // ...

  style: cytoscape.stylesheet()
    .selector('node')
      .style({
        'background-color': function( ele ){ return ele.data('bg') }

        // which works the same as

        // 'background-color': 'data(bg)'
      })

      // ...


  // , ...
});

                                                                                          属性类型

                                                                                          • 颜色可以通过名称（如指定red），十六进制（如#ff0000或#f00），RGB（例如rgb(255, 0, 0)），或HSL（例如hsl(0, 100%, 50%)）。
                                                                                          • 需要一个数字值，如长度，可在像素值（例如，指定24px），是隐含在像素（无单位的值24），或em值（例如2em）。大小中指定型号的坐标，所以屏幕上的（渲染）尺寸作为变焦1规定。
                                                                                          • 不透明度值被指定为序号为上0 <= opacity <= 1。
                                                                                          • 时间以ms或s为单位测量。
                                                                                          • 角度是弧度（如测量3.14rad）或度（例如180deg）。

                                                                                          映射器

                                                                                          除了直接指定属性的值之外，开发者还可以使用映射器来动态地指定属性值。

                                                                                          如果定义了映射，则定义所有元素的映射数据，或使用选择器将映射限制为定义了映射数据的元素。例如，选择器[foo]只适用于与数据字段的元素foo来定义。

                                                                                          • data()指定到元素的数据字段的直接映射。例如，data(descr)将一个元素的一个属性值映射descr在它的数据（即场ele.data("descr")）。这是映射像标签文本内容（属性有用的content属性）。
                                                                                          • mapData()指定到元素的数据字段的线性映射。例如，mapData(weight, 0, 100, blue, red)映射元素的重量之间的蓝色和红色为0和100之间的权重的元件与渐变ele.data("weight") === 0将被映射到蓝色，例如。其值落在指定范围之外的元素将映射到极值。在前面的例子中，用元素ele.data("weight") === -1将被映射到蓝色。
                                                                                          • function( ele ){ ... }函数可以作为样式属性的值传递。该函数有一个单一的ele，指定要为其计算的样式属性值的元素参数。该函数必须为其对应的选择器块应用的所有元素的相应样式属性指定有效的值。请注意，虽然方便，这些功能应该是廉价的来执行，其中像lodash的理想缓存_.memoize()。

                                                                                          节点体

                                                                                          • background-width：指定图像的宽度。中的百分比值（例如50%）可用于相对于该节点宽度图像宽度设置的。如果结合使用background-fit，则该值将覆盖在计算拟合所述图像的宽度-从而越过宽高比。的auto值是默认，它使用图像的宽度使用。
                                                                                          • background-height：指定图像的高度。中的百分比值（例如50%）可用于相对于该节点高度图象高度设置。如果结合使用background-fit，则该值将覆盖图像的高度在计算拟合-从而越过宽高比。的auto值是默认，它使用图像的高度使用。

                                                                                          饼图背景

                                                                                          这些属性允许您在节点上创建饼图背景。注意，16片最大是每个节点支持，因此在属性1 <= i <= 16。当然，你必须指定代替每个属性的数值i。每个非零尺寸切片被放置在顺序i，从12点的位置开始和工作顺时针方向转动。

                                                                                          您可能会发现为样式表中的所有节点保留特定颜色的数字很有用。然后，你可以指定值pie-i-background-size用于通过每个节点相应的映射。这将允许您基于元素数据在图的每个节点中创建一致的彩色饼图。

                                                                                          • pie-size：饼的直径，如节点的大小（例如的百分率来衡量100%）或绝对长度（如25px）。
                                                                                          • pie-i-background-color ：节点的第i个饼图切片的颜色。
                                                                                          • pie-i-background-size：节点的第i饼图片的大小，以百分比（例如测量25%或25）。
                                                                                          • pie-i-background-opacity ：节点的第i个饼图切片的不透明度。

                                                                                          边缘线

                                                                                          这些属性影响边线的样式：

                                                                                          • width ：边线的宽度。
                                                                                          • curve-style：用于分离两个节点之间的两个或更多个边的弯曲方法; 可以是haystack（默认，非常快，捆绑直边为哪些循环和化合物是不支持的）， bezier（捆绑弯曲的边缘）， unbundled-bezier（弯曲为与手动控制点使用边缘），或segments（一系列直线）。注意，haystack边缘与工作最好ellipse，rectangle或类似节点。小节点的形状，像triangle，会不会像美观。还要注意的是边缘箭不支持的haystack边缘。

                                                                                          • line-style：边线的样式; 可以是solid，dotted，或dashed。

                                                                                          贝塞尔边缘

                                                                                          对于自动捆绑贝塞尔边缘（curve-style: bezier）：

                                                                                          • control-point-step-size ：从垂直于源到目标的线，该值指定连续贝塞尔边缘之间的距离。
                                                                                          • control-point-distance：重写单个值control-point-step-size有手动值。因为它覆盖步长，具有相同值的贝塞尔边将重叠。因此，如果需要，最好将其用作特定边的一次性值。
                                                                                          • control-point-weight：一个单一值，它控制沿着从源到目标的线的点。该值通常范围为[0，1]，0表示源节点，1表示目标节点 - 但也可以使用更大或更小的值。
                                                                                          • edge-distances：与值intersection（默认值），从源线定位为control-point-weight是从源节点的形状的外侧到目标节点的形状的外侧。同值node-position，该行是从源位置到目标位置。该node-position选项使得计算的边缘点容易-但它应谨慎使用，因为你可以创建一个无效的点intersection会自动校正。

                                                                                          贝塞尔具有手动控制点的边缘（curve-style: unbundled-bezier）：

                                                                                          • control-point-distances：一系列指定为每个控制点垂直于从源形成的线到目标的距离，例如，值-20 20 -20。
                                                                                          • control-point-weights：一系列的权重控制沿着从源线的点值的目标，例如0.25 0.5 0.75。值通常范围为[0,1]，其中0朝向源节点，1朝向目标节点 - 但是也可以使用更大或更小的值。
                                                                                          • edge-distances：与值intersection（默认值），从源线定位为control-point-weights是从源节点的形状的外侧到目标节点的形状的外侧。同值node-position，该行是从源位置到目标位置。该node-position选项使得计算的边缘点容易-但它应谨慎使用，因为你可以创建一个无效的点intersection会自动校正。

                                                                                          • edge-distances：与值intersection（默认值），从源线定位为segment-weights是从源节点的形状的外侧到目标节点的形状的外侧。同值node-position，该行是从源位置到目标位置。该node-position选项使得计算的边缘点容易-但它应谨慎使用，因为你可以创建一个无效的点intersection会自动校正。

                                                                                          布局

                                                                                          布局的功能是设置图中节点上的位置。布局是扩展 Cytoscape.js的，使得它有可能为任何人，而无需修改库本身写的布局。

                                                                                          默认情况下，Cytoscape.js中包括几个布局，并且它们的选项在随后指定的默认值部分中描述。请注意，您必须设置options.name布局的名称来指定要运行哪一个。

                                                                                          cose

                                                                                          在cose（复合弹簧嵌器）的布局采用了物理模拟奠定了图表。它非常适合非复合图，它有额外的逻辑，以支持复合图。

                                                                                          它是由实施赫拉尔多·哈克为2013码的谷歌暑期的一部分（导师：最大弗朗茨，克里斯蒂安·洛佩斯，安德斯Riutta，的Ugur Dogrusoz）。

                                                                                          根据文章“无向图复合布局算法”通过的Ugur Dogrusoz，埃尔汗Giral体育场，艾哈迈德Cetintas，阿里奇夫里尔和埃梅克德米尔。

                                                                                          该cose布局是非常快的，并产生了良好的效果。该cose-bilkent扩展是算法是计算上更昂贵，但产生近乎完美的结果的演化。

                                                                                          选项

                                                                                          var options = {
  name: 'cose',

  // Called on `layoutready`
  ready: function(){},

  // Called on `layoutstop`
  stop: function(){},

  // Whether to animate while running the layout
  animate: true,

  // The layout animates only after this many milliseconds
  // (prevents flashing on fast runs)
  animationThreshold: 250,

  // Number of iterations between consecutive screen positions update
  // (0 -> only updated on the end)
  refresh: 20,

  // Whether to fit the network view after when done
  fit: true,

  // Padding on fit
  padding: 30,

  // Constrain layout bounds; { x1, y1, x2, y2 } or { x1, y1, w, h }
  boundingBox: undefined,

  // Randomize the initial positions of the nodes (true) or use existing positions (false)
  randomize: false,

  // Extra spacing between components in non-compound graphs
  componentSpacing: 100,

  // Node repulsion (non overlapping) multiplier
  nodeRepulsion: function( node ){ return 400000; },

  // Node repulsion (overlapping) multiplier
  nodeOverlap: 10,

  // Ideal edge (non nested) length
  idealEdgeLength: function( edge ){ return 10; },

  // Divisor to compute edge forces
  edgeElasticity: function( edge ){ return 100; },

  // Nesting factor (multiplier) to compute ideal edge length for nested edges
  nestingFactor: 5,

  // Gravity force (constant)
  gravity: 80,

  // Maximum number of iterations to perform
  numIter: 1000,

  // Initial temperature (maximum node displacement)
  initialTemp: 200,

  // Cooling factor (how the temperature is reduced between consecutive iterations
  coolingFactor: 0.95,

  // Lower temperature threshold (below this point the layout will end)
  minTemp: 1.0,

  // Whether to use threading to speed up the layout
  useMultitasking: true

                                                                                                    多任务API被内置到Cytoscape.js中，用于布局等扩展，例如使布局更快。这些API是从拉入韦弗库，把对cytoscape对象，而不是weaver。例如，您可以通过让一个线程cytoscape.thread()，而不是通常的weaver.thread()。

                                                                                                    性能

                                                                                                    背景

                                                                                                    您可能会注意到，在具有大量元素的图形上，性能开始下降。这是由于以下几个原因：

                                                                                                    • 性能是图大小的函数，因此性能随着元素数量的增加而减少。
                                                                                                    • Cytoscape.js支持的丰富的视觉风格可能很贵。只画圈和直线很便宜，但绘制复杂的图不那么简单。
                                                                                                    • 边缘渲染特别昂贵。多边形变得甚至更昂贵，需要贝塞尔曲线边缘。
                                                                                                    • 渲染（位图）画布的性能是它需要渲染的区域的函数。因此，增加的像素比率（如在高密度显示器中，如在iPad上）可以降低渲染性能。

                                                                                                    优化

                                                                                                    您可以通过调整您的选项，以显着性的降序从Cytoscape.js获得更好的性能：

                                                                                                    • 批处理元素的修改：使用cy.batch()一次修改的元素。
                                                                                                    • 动画：您将获得无动画更好的性能。如果使用动画：
                                                                                                      • eles.flashClass() 是一个比平滑动画更便宜的替代品。
                                                                                                      • 尝试限制并发动画元素的数量。
                                                                                                      • 当在使用风格过渡动画，确保transition-property只为要动画状态定义。如果您transition-property在默认状态下定义，动画将尽量多比，如果你限制它你真正想要的动画特定国家运行。
                                                                                                    • 功能样式属性值：虽然方便，功能样式属性值可以是昂贵的。因此，它可能是值得的，如果可以使用缓存，诸如通过使用lodash _.memoize()功能。如果你的样式属性值是一个简单的直通或线性映射，请考虑使用data()或mapData()代替。
                                                                                                    • 标签：拉丝标签是昂贵的。
                                                                                                      • 如果你可以去没有他们或显示他们tap / mouseover，你会得到更好的性能。
                                                                                                      • 考虑不要为边设置标签。
                                                                                                      • 考虑设置min-zoomed-font-size这样当标签是小你的风格-硬反正看-他们不渲染。当标签至少是您设置的大小（即用户放大）时，它们将可见。
                                                                                                      • 背景和轮廓增加了渲染标签的费用。
                                                                                                    • 简化边风格：拉丝边可以是昂贵的。
                                                                                                      • 您的边缘设置curve-style为haystack在样式表。草堆边缘是直线，这是很大的呈现比较便宜bezier的边缘。这是默认边缘样式。
                                                                                                      • 阴影非常昂贵。如果性能成为问题，请避免使用。
                                                                                                      • 使用实边。虚线和虚线边缘的绘制要贵得多，因此不使用它们可以提高性能。
                                                                                                      • 边缘箭头的呈现代价很高，因此如果图形中没有任何语义，请考虑不使用它们。
                                                                                                      • 带箭头的不透明边缘比具有箭头的半透明边缘快两倍。
                                                                                                    • 简化节点风格：为节点某些风格可以是昂贵的。
                                                                                                      • 阴影非常昂贵。如果性能成为问题，请避免使用。
                                                                                                      • 背景图像在某些情况下非常昂贵。最高效的背景图像被非重复（background-repeat: no-repeat）和非削波（background-clip: none）。对于简单的节点形状，如正方形或圆形，则可以使用background-fit缩放和（如与preclip您的图像模拟软件裁剪咕嘟咕嘟所以它的自动化）。代替预剪切，您可以聪明地使用具有透明背景的PNG。
                                                                                                      • 节点边界可能会稍微昂贵，因此您可以尝试删除它们，看看它是否对您的用例产生显着的差异。
                                                                                                    • 设置较低的像素比例：因为它是更昂贵的渲染更多的像素，可以设置pixelRatio为1 在初始化选项，以增加对高密度显示屏大型图形性能。然而，这使得渲染不那么脆。
                                                                                                    • 复合节点：复合节点使风格的计算和渲染更加昂贵。如果图不需要复合节点，则可以通过不使用复合父节点来提高性能。
                                                                                                    • 隐藏在互动边缘：设置hideEdgesOnViewport为true你的初始化选项。这可以通过在平移，鼠标滚轮缩放，缩放到缩放和节点拖动动作期间隐藏边缘，使非常大的图形的交互性更便宜。此选项只对非常大的图形产生影响。
                                                                                                    • 回收大型实例：大型实例可以使用大量的内存，主要是由于帆布使用。循环的竞争的实例将有助于保持你的内存使用率不是要求低cy.destroy()，因为你不会增加堆一样多，你不会尽可能多的调用垃圾收集器。
                                                                                                    • 使用纹理缩放和平移：设置textureOnViewport为true你的初始化选项。不是重新渲染整个场景，而是在平移和缩放操作开始时创建视口的纹理缓存，而不是对其进行操作。使非常大的图形的平移和缩放更平滑。此选项只对非常大的图形产生影响。该渲染器拥有更多质地一般缓存支持，所以textureOnViewport只有有用的，如果你真的需要绝对最便宜的选择。
